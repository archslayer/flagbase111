# Quest Free-Attack Fix Report

## Scope
- `lib/processAttackEvent.ts` (new helper)
- `workers/attack-events.worker.ts`
- `app/api/queue/attack-events/route.ts`
- `app/api/attacks/verify/route.ts` (new endpoint)
- `app/api/quests/claim/route.ts`
- `app/quests/page.tsx`

## DB Assumptions
- Collections `free_attacks` and `processed_tx` must have unique indexes on `wallet` and `txHash` respectively (`createIndex({ wallet: 1 }, { unique: true })`, `createIndex({ txHash: 1 }, { unique: true })`).
- `free_attacks` documents follow `{ wallet, awarded, used, totalLimit }`, with default `totalLimit = MAX_FREE_ATTACKS_PER_USER (2)`.

## Fixes

### 1. Atomic Consumption Helper
- Added `lib/processAttackEvent.ts` implementing idempotent, atomic consumption of free attacks.
- Uses `$expr` to ensure `used < min(awarded, totalLimit)` before incrementing, preventing race conditions.
- Records results in `processed_tx` to skip duplicate tx hashes.

### 2. Queue & Worker Integration
- `workers/attack-events.worker.ts` now delegates consumption to the helper, logging outcomes.
- `/api/queue/attack-events` processes consumption immediately after enqueue (try/catch), so even if the worker lags the DB is updated.

### 3. Manual Verification Endpoint
- New `POST /api/attacks/verify` route calls `processAttackEvent` synchronously. Frontend hits this right after an on-chain success to guarantee consumption even if the queue is slow.

### 4. Quest Claim Upsert
- `/api/quests/claim` normalizes `free_attacks`: creates documents when missing, increments `awarded` up to `totalLimit`, and ensures `used` defaults to 0. Re-claims simply ensure the document exists instead of bumping `awarded` past the cap.

### 5. Frontend Guard Rails
- The quest attack modal now:
  - Optimistically decrements remaining/used counts on success.
  - Calls `/api/attacks/verify` with `{ txHash, user, feeUSDC6: 0 }` and waits for confirmation.
  - Refetches `/api/free-attack/my` to sync with backend; rolls back via refetch if verification fails.
  - Keeps the modal disabled until verification finishes; victory splash closes the modal when dismissed.

## Testing Performed
- `pnpm lint` (via `read_lints`) on touched files – no issues.
- Manual DB probe before/after attack (using a Node script) to confirm `free_attacks` documents update correctly when awards are present.
- Verified claim path produces/updates `free_attacks` entries even when quest was completed earlier.
